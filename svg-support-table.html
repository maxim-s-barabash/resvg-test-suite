About

This page contains results of the resvg test suite which is specifically designed for the resvg library.

Unlike an official SVG test suite this one is focused on edge-cases.

Major differences:

  • More edge-cases.

  • One file - one test.
    In the official test suite, one file can have a lot of tests.

  • Only supported SVG subset.
    resvg trying to support only the static SVG subset. So this test suite doesn’t have any animation and scripting tests.

  • No embedded fonts.
    We don’t plan to support them. So there are no tests for now.

  • No raster images as references.
    We are using vdiff tool to check that test is passed.

  • No percent values in width and height.
    Isn’t supported that good.


We are testing/comparing resvg with Chrome, Inkscape, librsvg and QtSvg for a reason.

  • Chrome has the best SVG support and used as a reference implementation.

  • Inkscape is an obvious choice on Linux.

  • librsvg is the main resvg rival. Used in GNOME, XFCE for SVG rendering. The goal is to replace librsvg with resvg.

  • QtSvg is used by all the Qt applications and KDE. We plan to replace it as well.

Naming convention: type-name-###.svg

type can be either a(attribute) or e(element).

For example the polyline - 3. Ignore odd points will lead to e-polyline-003.svg.
